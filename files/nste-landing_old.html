Environment Rules

1
24-hour max booking. Sessions auto-destroy at expiry. Extend via ServiceNow before time runs out.
2
4-hour idle timeout. Inactive topologies are automatically torn down to free resources.
3
Save before you leave. Export configs from running nodes before your session ends — all state is destroyed on teardown.
4
One booking at a time. The environment supports a single concurrent user. Check availability before booking.
5
No production data. Do not load production configs, credentials or sensitive data into the test environment.
6
Report issues. Log faults via the NSTE support channel — don't attempt to fix platform-level issues yourself.
